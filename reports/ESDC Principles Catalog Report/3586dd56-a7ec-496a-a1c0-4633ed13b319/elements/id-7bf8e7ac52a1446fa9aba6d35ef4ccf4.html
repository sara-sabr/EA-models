6.2. Responsive Change Management Name: Responsive Change Management Statement: Changes to the enterprise information/data and technology environment are implemented in a timely manner Rationale: • If people are to be expected to work within the enterprise information /data and technology environment, this environment must be responsive to their needs Implications: • Processes for managing and implementing change must be developed that do not create delays • A user who feels a need for change will need to connect with a “business expert” to facilitate explanation and implementation of that need • If changes are going to be made, the architecture must be kept updated References: • TOGAF 9.2 Principle 19
